# How Pages Work

Use the Page Builder for regular pages.

Only do not use it if you are:

- editing an existing page by hand
- making a custom page with special JavaScript or layout
- making something like a gallery, builder, tool app, or other non-standard page

## Core Rule

If the page is a normal tool, mod, guide, patch, plugin, or utility page, use `public/pagebuilder.html`.

That is the default way to make pages for this site.

## What Is Core

These are the normal site categories used on `public/index.html`.

### Core / Main Categories

- `Most Popular & Essential Tools`
- `Blocking Amazon`
- `KOReader Plugins`
- `KOReader Patches`
- `Other KOReader`
- `Emulators`
- `Media & Audio`
- `Drawing Tools`
- `Tools`
- `Resources & Utilities`
- `Development`
- `Informational`
- `Beta & Advanced`

These are the normal homepage sections. Most new pages should fit into one of these.

## What "Other Cool Stuff" Is For

`Other Cool Stuff` is not the main place for normal pages.

Use `Other Cool Stuff` only for things that are:

- unusual
- experimental
- untested
- risky
- hard to categorize cleanly
- more like a tip, oddity, external link, or advanced niche tweak than a full normal tool page

Do not put normal tools there if they fit a real homepage category.

## What Kind of Pages Use the Builder

Use the Page Builder for pages like:

- one tool
- one mod
- one patch
- one plugin
- one utility
- one guide
- one informational page

These are the normal single-subject pages on the site.

## What Kind of Pages Usually Do Not Use the Builder

Usually do not use the Page Builder for:

- `public/index.html`
- image galleries
- interactive apps
- editors
- pages with custom scripts and custom UI
- aggregator pages with many unrelated items

Examples of more custom pages:

- `public/index.html`
- `public/images.html`
- `public/pagebuilder.html`
- `public/editor.html`

## Simple Rule for Contributors

When in doubt:

1. use the Page Builder
2. export the page
3. add the homepage card if it belongs on the homepage
4. only hand-edit if the page is custom or unusual

## Footer Rule

Every normal page should end with the standard legal footer:

> Educational purposes only. Not affiliated with Amazon. Users responsible for compliance with applicable laws.

## Short Version

- normal page = use Page Builder
- custom page = hand edit
- main content belongs in the normal homepage categories
- weird / risky / experimental content may go in `Other Cool Stuff`
